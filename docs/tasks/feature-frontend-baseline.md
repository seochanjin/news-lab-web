# Task: NewsLab Web 프론트엔드 MVP 레포 구성

## Goal

NewsLab 백엔드 API를 사용하는 프론트엔드 애플리케이션의 초기 레포 구조를 구성한다.

이번 작업의 목표는 완성형 UI가 아니라, Next.js 기반 프론트엔드 개발을 시작할 수 있는 최소 구조, 검증 명령, CI, agent workflow 문서 기반을 마련하는 것이다.

## Scope

- Next.js + TypeScript + Tailwind CSS 기본 프로젝트를 유지한다.
- Node.js 버전을 `.node-version`으로 고정한다.
- 프론트 전용 `AGENTS.md`를 작성한다.
- `docs/tasks`, `docs/pr`, `docs/devlog`, `docs/verification`, `docs/reviews`, `docs/fixes` 구조를 사용한다.
- GitHub Actions CI를 추가한다.
- CI는 install, lint, typecheck, build를 수행한다.
- API base URL은 환경변수로 관리하도록 설계한다.
- `.env.example`을 추가한다.
- 기본 홈 화면을 NewsLab Web placeholder로 정리한다.
- README를 NewsLab Web 기준으로 정리한다.

## Do not change

- Do not modify the backend repository.
- Do not modify K3s, Supabase, database, or production infrastructure.
- Do not store secrets, tokens, credentials, `.env` values, or private keys.
- Do not implement production deployment in this task.
- Do not add Docker/K3s deployment yet.
- Do not run git push or PR merge.

- Next.js app exists and builds successfully.
- TypeScript is enabled.
- Tailwind CSS is enabled.
- .node-version exists.
- npm run lint passes.
- npm run typecheck passes.
- npm run build passes.
- GitHub Actions CI exists and runs install, lint, typecheck, and build.
- .env.example documents NEXT_PUBLIC_NEWSLAB_API_BASE_URL without storing real secrets.
- AGENTS.md contains frontend-specific rules while preserving the Next.js generated agent warning.
- README explains local setup and validation commands.
- Backend/infra repository is not modified.
